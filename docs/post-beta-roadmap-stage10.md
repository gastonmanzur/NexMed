# NexMed — Roadmap inicial post-beta (Etapa 10)

## Horizonte inmediato (0-2 semanas)
- Resolver P0 de estabilidad y permisos.
- Resolver bugs P1 de turnos y disponibilidad.
- Ajustes UX rápidos en formularios críticos.

## Siguiente sprint (2-6 semanas)
- Mejoras operativas priorizadas por feedback recurrente.
- Hardening de notificaciones y mensajes de error.
- Refinar onboarding y configuración según fricciones detectadas.

## Horizonte estratégico (6-12 semanas)
- Features futuras con validación real de demanda.
- Métricas de producto incrementales (sin sobreingeniería).
- Preparación de apertura gradual a más organizaciones.

## Puerta de expansión
Abrir beta ampliada solo si:
- no hay bloqueantes abiertos,
- backlog P0/P1 está controlado,
- experiencia base es estable para centro y paciente.
